# Thesis Progress Update — June 25, 2026

## Project Goal

Build a framework that systematically tests and challenges the assumptions made by federated learning defense strategies. The framework uses adversarial attack sweeps, adaptive attack selection, and structured logging to discover defense-specific weaknesses — framed using MITRE ATLAS threat categories.

---

## What Has Been Implemented

### 1. Four New Defense Strategies (server_app.py)

Added FoolsGold, FLRAM, MAB-RFL, and expanded FLTrust — bringing the total to 8 testable strategies:

| Strategy | Type | Status |
|----------|------|--------|
| FedAvg | Baseline | Existed |
| Bulyan | Byzantine-robust | Existed |
| MultiKrum | Byzantine-robust | Existed |
| FedTrimmedAvg | Byzantine-robust | Existed |
| FedMedian | Byzantine-robust | Existed |
| FLTrust | Trust-based | **Expanded** — added configurable trust strength, min weight, warmup rounds |
| FoolsGold | Trust-based | **New** — historical update-similarity trust weighting |
| FLRAM | Trust-based | **New** — credibility scoring via norm, direction, and sign agreement |
| MAB-RFL | Trust-based | **New** — reputation/bandit-style adaptive trust weighting |

All four trust-based strategies:
- Log per-client trust scores, selection decisions, norms, cosine similarity, and reputation to `trust_strategy_by_round.csv`
- Support configurable trust strength via `--trust-level` (weak/medium/full)
- Integrate with the existing attack injection mixin so adversarial sweeps work automatically

### 2. Trust Logging Infrastructure (server_app.py)

Built a unified trust logging system in the `AttackInjectedStrategyMixin`:
- Every trust strategy writes per-client, per-round rows to a structured CSV
- Columns: round, strategy, client_id, trust_score, selected_for_aggregation, update_norm, cosine_to_center, history_score, reputation, num_examples, details_json
- The `details_json` field captures strategy-specific internals (semicolon-escaped for CSV safety)
- This feeds directly into the database schema for automated vulnerability analysis

### 3. Configurable Trust Strength (pyproject.toml + run_thesis_sweep.sh)

Added three trust-level presets controllable from the command line:

| Level | trust-strength | min-weight | warmup-rounds | Purpose |
|-------|---------------|------------|---------------|---------|
| weak | 0.15 | 0.75 | 5 | Barely active — tests if trust adds value at all |
| medium | 0.6 | 0.3 | 3 | Moderate filtering |
| full | 1.0 | 0.0 | 3 | Maximum trust enforcement — intended for stress testing |

This is toggled via `--trust-level weak|medium|full` in the sweep script.

### 4. Expanded Metrics (client_app.py + server_app.py)

Pilot runs now produce:
- Accuracy, loss
- F1 (macro + weighted)
- Precision (macro + weighted)
- Recall (macro + weighted)
- Per-class accuracy (62 classes for FEMNIST)
- Backdoor ASR + backdoor loss
- Per-client trust scores and selection decisions (trust strategies only)

### 5. Reproducibility Improvements (client_app.py)

- Global seed propagation: random, NumPy, and PyTorch seeds are now set in both `train()` and `evaluate()` from the attack-seed config
- Boolean config parsing hardened to handle string/int/float edge cases from TOML overrides

### 6. Sweep Infrastructure Expansion (run_thesis_sweep.sh)

- Strategy normalization handles all 8 strategies (including aliases like `fools-gold`, `mab_rfl`)
- `--trust-level` flag added
- Default strategy list expanded from 3 to all 8
- Clean baseline runs automatically generated for each strategy

### 8. Database Schema for Vulnerability Discovery (db/)

Designed and validated (with dummy data) a 13-table SQLite schema for structured analysis:
- Stores sweeps, runs, configs, round-level metrics, client metrics, attack events, trust scores, defense selections, and baseline comparisons
- Supports agent-queryable vulnerability discovery patterns (slipthrough rate, adaptive convergence, collapse detection, etc.)
- 11 vulnerability discovery query functions implemented and tested against dummy data
- Schema documented in `docs/DATABASE_WORKFLOW.md` with full CSV-to-table mapping

### 9. LLM Analysis Pipeline (scripts/llm_sweep_analysis.py)

Extended the LLM sweep analysis script to summarize trust strategy logs alongside attack timelines and metrics, producing per-strategy and global vulnerability reports.

## Architecture Summary

```
pyproject.toml          <- all experiment parameters, toggleable per-run
run_thesis_sweep.sh     <- iterates strategies x scenarios x seeds
  -> scripts/run_simulation_and_log.py   <- runs one experiment, saves all artifacts
    -> pytorchexample/
        task.py         <- model, datasets, AttackEngine, all attack types
        server_app.py   <- 8 aggregation strategies + trust logging
        client_app.py   <- training loop with attack injection
  -> logs/sweeps/       <- structured output per run
  -> db/                <- SQLite schema + queries for vulnerability analysis
  -> dashboard/         <- web UI for monitoring (functional, lower priority)
```
